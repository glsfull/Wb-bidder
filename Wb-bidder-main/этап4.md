# Этап 4. Рабочий экран биддера

## Что сделано

- Расширен личный кабинет Nuxt 3 в `frontend/app/pages/dashboard.vue` до рабочего экрана биддера.
- Добавлена верхняя панель магазина: выбор магазина, период, статус синхронизации, баланс и ручное обновление.
- Добавлены фильтры кампаний по статусу и типу, а также отображение стратегии по каждой кампании.
- Добавлена карточка товара с артикулом WB, vendor code, бюджетом, расходом и заказами.
- Добавлены зоны показа "Поиск" и "Рекомендации" с inline-редактированием ставок и подсказкой про ограничение единой ставки для автоматических кампаний.
- Добавлены ограничения биддера: минимальная ставка, максимальная ставка, шаг изменения и дневной лимит.
- Добавлена таблица фраз и кластеров с KPI: статус, позиция, частота, показы, клики, CTR, расход, CPC, заказы, CPO, ДРР, текущая ставка, рекомендуемая ставка и действие.
- Добавлены массовые действия: изменение ставок, установка лимитов, остановка убыточных фраз, применение рекомендаций и экспорт CSV.
- Добавлены состояния интерфейса: загрузка, нет кампаний, истекший токен, частичная ошибка API, неподдерживаемые раздельные ставки, выключенный биддер и запущенная очередь обновления.
- Расширены стили в `frontend/assets/css/main.css`, чтобы экран оставался читаемым на desktop и mobile.
- Добавлен тестовый список требований Phase 4 в `frontend/tests/app-shell.test.ts`.

## Проверка на ошибки

Локально нужно запускать проверки из папки `frontend`:

```bash
npm install
npm run test
npm run lint
npm run build
```

Если зависимости уже установлены, достаточно:

```bash
cd frontend
npm run test
npm run lint
npm run build
```

## Как запустить проект сайта с GitHub для визуального просмотра

1. Клонировать репозиторий и перейти в ветку этапа:

```bash
git clone https://github.com/glsfull/Wb-bidder.git
cd Wb-bidder
git checkout issue-7-93c12e88ed5f
```

2. Установить зависимости frontend:

```bash
cd frontend
npm install
```

3. Запустить сайт в режиме разработки:

```bash
npm run dev
```

4. Открыть в браузере адрес, который покажет Nuxt. Обычно это:

```text
http://localhost:3000
```

5. Для визуального просмотра рабочего экрана перейти на:

```text
http://localhost:3000/dashboard
```
